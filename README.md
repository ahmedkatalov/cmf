# Frontend Onboarding — Finance App (Auth + Branches + Users)

Привет! Это инструкция по подключению фронта к готовому backend API (регистрация/логин + точки + пользователи).  
Backend уже реализован и задеплоен/залит в GitHub.

---

## 1) Контекст проекта (важно для понимания UI/логики)

У нас есть модель:

- **Organization (Организация/Компания)** — создаётся один раз при регистрации Owner.
- **Branch (Точка/Филиал/Офис)** — создаётся Owner’ом (главным аккаунтом).
- **User (Пользователь)** — принадлежит Organization и конкретной Branch.

### Роли и доступ:

#### ✅ Owner (владелец)
Owner — главный аккаунт. Может:
- регистрировать организацию
- создавать точки (branches)
- создавать админов точек
- создавать сотрудников для любой точки
- позже видеть отчёты по всем точкам + общий итог

#### ✅ Admin (админ точки)
Admin — админ конкретной точки. Может:
- создавать пользователей (сотрудников) только внутри своей точки
- видеть только данные своей точки (клиенты/операции/доходы)
НЕ может:
- создавать точки
- создавать других admin

#### Другие роли (пока только заводим аккаунты)
- `manager`
- `accountant`
- `security`
- `employee`
